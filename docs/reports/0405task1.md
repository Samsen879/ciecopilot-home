## Step 3：最小实现修改

## A. `error-contract.js` 的修改

### 1. 收窄错误码集合
移除了不在本次 frozen stage codes 内的额外代码：
- `REVIEW_TASK_NOT_FOUND`
- `REVIEW_TASK_STATE_CONFLICT`
- `SUBJECT_ADAPTER_NOT_ENABLED`

### 2. 同步收窄错误定义表
删除了上述 3 个 code 对应的 definition。

### 3. 加入深拷贝与深冻结
新增：
- `cloneValue`
- `deepFreeze`

使 `buildLearningError(...)` 返回：
- 顶层对象冻结
- `error` 对象冻结
- `details` 深拷贝后冻结

### 4. 结果
`buildLearningError` 现在满足：
- frozen envelope shape
- 不直接复用外部传入 `details` 引用
- 返回结构稳定

---

## B. `released-scope.js` 的修改

### 1. 将 seeded pilot membership 收窄到 frozen trig slice
新增逻辑从 released-family gate receipt 加载 family 数据，并只认：

- `9709.trigonometry_manipulation_equations`

作为 frozen pilot family。

### 2. 从 registry 派生允许的 pilot types
从该 family 的 `released_question_type_ids` 派生 membership，而不是沿用“任何 released type 都算 pilot”这一更宽行为。

因此本次收敛后：
- `9709.trigonometry.identities` → true
- `9709.trigonometry.equations` → true
- `9709.integration.application` → false
- `9709.differential_equations.separable` → false

### 3. 调整 posture helper 的 pilot 判定
通过包装 `resolveInlineReleasedScoringPosture` / `resolveReleasedScoringPosture`，确保：

- authoritative scoring 的前提中，pilot membership 必须来自本次 frozen trig slice
- 非 pilot released family 仍然 fallback
- pilot membership alone 仍然不能独立解锁 scoring

### 4. 结果
`released-scope.js` 现在仍然是：
- registry-backed
- 带 posture helper

但行为已收窄到本次任务要求的 frozen pilot 范围。
